Aufgabe 1.2

"HTTP"

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
    2.
    • Die Anfrage war erfolgreich.
  2. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
    3.
    • Weiterleitung von einer URL zu einer anderen.
  3. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
    • 400 Bad Request => Server ist nicht in der Lage, die Anfrage zu bearbeiten. Mögliche gründe: Falsche Webadresse, Probleme mit Cookies oder zu Lange URLs. Man kann in dem Fall die Anfrage überprüfen, ob sie korrekt ist und ob irgendwelche Infos fehlen.
  4. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
    • 403 Forbidden, erforderliche Rechte für den Zugriff auf die Ressourcen fehlen.
  5. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
    • Man kann die Anfrage cacheable machen, indem man den http-Caching-Header verwendet. Man kann die Berechtigung, die IP-Einschränkung und die Server-Logs überprüfen.
  6. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performancegründen soll die Abfrage aber cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

Aufgabe 1.6

"HTML-Literatur lesen und Fragen beantworten"

  1. Was ist HTML?
    2.
    • HTML steht für "Hypertext Markup Language" und ist eine standardisierte Auszeichnungssprache, die zur Erstellung von Webseiten und Webanwendungen verwendet wird.
  2. Wie kann man eine geschachtelte geordnete Liste der Tiefe 3 erzeugen?
    3.
    • <ol><ol><ol><li></li></ol></ol></ol>
  3. Wie ist eine HTML-Tabelle aufgebaut?
    • <table><tr><td></td></tr></table>
  4. Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?
    • Man sollte Leerzeichen und Sonderzeichen vermeiden, aussagekräftige Namen und Kleinbuchstaben verwenden und Ordner gut strukturieren.
  5. Wie baut man in HTML ein Menü?
    • Z.B. mit einer Liste und hyperlinks
      • Home
      • Page1
      • Page2
  6. Welche Attribute sollte man bei Bildern wie verwenden?
    • src (Quelle): Gibt den Dateipfad oder die URL zum Bild an.
    • alt (Alternativtext): Beschreibt das Bild für Barrierefreiheit und SEO.
    • title (Titel): Zeigt einen Tooltip-Text beim Überfahren des Bildes mit der Maus.
    • width (Breite) und height (Höhe): Definieren die genaue Bildgröße in Pixeln.
