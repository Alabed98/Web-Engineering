10.1 Static classes in TypeScript

Gibt es in TypeScript statische Klassen? Erläutern Sie den Unterschied von TypeScript zu anderen objekt-orientierten, statisch typisierten Programmiersprachen wie Java und C#.

Meine Lösung:

In TypeScript gibt es keine direkten statischen Klassen im Vergleich zu anderen objektorientierten

TypeScript verwendet eine optionale statische Typisierung, was bedeutet, dass mann Typen für Variablen und Funktionen optional deklarieren kann.

TypeScript-Code wird in JavaScript transpiliert und läuft in einer JavaScript-Laufzeitumgebung. Im Gegensatz dazu werden Java- und C#-Code in Bytecode übersetzt und in einer eigenen virtuellen Maschine ausgeführt

10.2 Fehler finden mit TypeScript

Gegeben sei folgendes fehlerbehaftete JavaScript-Programm. Ergänzen Sie dieses um möglichst viele Typisierungen und übersetzen Sie es in ein TypeScript-Programm.

const add = (x,y) => x+y;

const equals = (x,y) => x===y;

console.log( add(1,2) );

console.log( add(true, true ) );

console.log( add(true, false ) );

var x = y = 1;

console.log( add(equals(x,y), equals(y,x)) );

Meine Lösung:

Typfehler können im JavaScript-Code in den Funktionen add und equals auftreten, wenn die Funktionen mit unterschiedlichen Datentypen aufgerufen werden.

TypeScript kann Typfehler besser erkennen, indem man explizite Typen für die Funktionen deklariert.

Der Code sieht dann in TypeScript so aus:

  • const add = (x: number, y: number): number => x + y;
  • const equals = (x: any, y: any): boolean => x === y;
  • console.log(add(1, 2));
  • console.log(add(true, true));
  • console.log(add(true, false));
  • var x = y = 1;
    • console.log(add(equals(x, y), equals(y, x)));

Weitere Beispiele für TypeScript-Hilfen bei der Fehlersuche:

  1. Verhindern von unbeabsichtigter Verwendung von undefinierten Variablen.
  2. Früherkennung von Tippfehlern durch statische Typprüfung.
